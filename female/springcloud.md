# 架构演变
1. **单体应用架构**
    - 维护苦难
    - 紧密耦合，单点容错低
    - 无法针对某个模块水平扩展
2. **垂直应用架构**
    - 模块之间无法调用
3. **分布式架构**
    - 调用关系复杂，维护困难
4. **SOA架构-面向服务的架构**
    - 解决复杂调用关系-服务治理中心-Dubbo
    - 服务依赖性 拆分不彻底
5. **微服务架构(服务原子化拆分)**
    - 服务原子化拆分，独立打包、部署、升级
    - 服务之间采用RestFul等轻量级http协议互相调用
    - 分布式系统开发的技术成本高
    - 服务注册、服务发现、服务剔除
    - ServiceComb、SpringCloud、SpringCloud Alibaba
